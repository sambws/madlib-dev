MADLIB

Madlib is an in-development kit that has built-in shorthand functions for Love2D, and uses Moonscript as it's language of choice to make code pretty and simple.

API

Madlib, in basic terms, is a Lua table with a bunch of useful functions inside. Some of them, such as the gamepad portion of the library, are simply wrapper functions designed to make writing out gamepad functionality in your code easier and more fluid. Others, such as the entity system, is a multitude of functions designed for an easy and lightweight object-oriented entity system. See the REFERENCE portion of this site for more information on the functionality of Madlib.

REFERENCE (HUGE WORK IN PROGRESS)

This portion of the site is dedicated to running through the functions of Madlib, and will serve as it's official documentation.

ENTITIES

The Entity class is the base class that all game objects should extend. For now, it ships with automatic positioning and z-ordering functions in it's step and initialize functions, but will most likely be fleshed out in the future. Use Moonscript's super function to utilize the Entity class to it's fullest extent.

addEnt(e) is a function used to create entities and add them to the ents table, thus rendering and updating it. The only argument it takes is used to define the entity you wish to create.
Example: addEnt(Box(32, 64)) would add a new Box entity at an x position of 32 and a y position of 64, if it's entity super is set up properly.

destroySelf(e) is a function that is called within entities to remove themselves from the ents table, subsequently stopping all rendering/updating of that game object.

zord(s, mod) is a function that is called within the base of the Entity class (so you most likely won't need to call this unless you're working on an entity that is not an Entity). It sets the game object's Z coordinate to -Y, meaning the negative Y position of the object.

ROOMS

Rooms in Madlib are handled in a very particular way. You register rooms in main.moon, meaning you fill out a table value with it's own name variable and event function. The name variable is used in switching and running the room, while event is filled with the code that is run when the room is switched to. If you're used to Game Maker, think of the event function to be akin to the creation code of each Game Maker room.

switchRoom(r) is used to switch the room. The 'r' argument refers to the name of the room you would to switch to. When rooms are switched, the room's event is ran, creating all game objects.

@pers is a variable that is optional to fill out in the new: function of entities. If you set it to true, it won't be deleted when the room is switched, making it persistent throughout all rooms. This is handy for things like controllers and backgrounds.

INPUT

Input functions in Madlib are very basic, and mainly exist to make writing code faster. It has functions for use with both controllers and keyboards.

key(k) will return true if a certain key is being pressed, with the key being the k variable. See Love2D's KeyConstant section for more info on which buttons to check for.

joyButton(c, b) is a function that will return true if a certain button (b) is being pressed on a certain controller (c). See LOVE2D's GamepadButton section for more info on which buttons to check for.

joyAxis(c, a) will return the value of a certain axis (a) on a certain controller (c).

joyConnected(c) will return true if a certain controller (c) is connected, and false if it isn't
